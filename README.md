<h2 align='center'>Cadastro de Peças Eletrônicas.</h2>

<br>


# Objetivo

Sistema desktop desenvolvido para meu pai um técnico em eletrônica com o objetivo de organizar o estoque de peças e reduzir perdas por falta de controle.

# Problema

O controle manual dificultava a localização de peças e aumentava o risco de perda de componentes importantes durante os atendimentos.

# Solução

A aplicação centraliza o controle de peças, permitindo cadastro, atualização de estoque e busca rápida, reduzindo o tempo gasto na organização e melhorando a eficiência no dia a dia.

# Decisões técnicas

* Uso de Java Swing para aplicação leve e offline.
* SQLite para persistência simples sem necessidade de servidor
* Estrutura pensada para futura expansão

# O que aprendi

* Estruturação de um CRUD completo com persistência em Java e SQLite
* Separação de responsabilidades entre interface, dados e model
* Desafios ao trabalhar com Java Swing (como manutenção de estado e eventos)
* Primeira experiência construindo algo para um usuário real.
* Geração do executável com bibliotecas necessárias.

# Limitações

* Interface simples, focada em funcionalidade
* Sem autenticação de usuários
* Não possui controle avançado de estoque

# Tecnologias

* Java
* Java Swing
* SQLite
* rs2xml
